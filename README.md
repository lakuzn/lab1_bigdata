# lab1_bigdata
Этот проект представляет собой микросервисную архитектуру для потоковой обработки заявок в службу технической поддержки (service-desk). Система симулирует поступление тикетов в реальном времени, передает их через брокер сообщений Kafka и на лету предсказывает приоритет заявки с помощью модели машинного обучения.

## Структура проекта
* `backend/` — скрипты `producer.py` (отправка сырых данных) и `consumer.py` (получение данных и ML-предикты).
* `frontend/` — аналитический дашборд на базе Streamlit (`app.py`).
* `data/` — генерация датасета (`service_desk_tickets.csv`) и файл с результатами для дэшборда.
* `notebooks/` — обученная модель и энкодеры (в формате `.pkl`).
* `docker-compose.yml` — конфигурация для развертывания всей инфраструктуры.

## Запуск проекта
1. Откройте терминал и перейдите в корневую папку проекта.
2. Выполните команду для сборки образов и фонового запуска всех контейнеров:
   docker-compose up --build -d

## Просмотр результатов
Как только контейнеры успешно запустятся, интерактивный дашборд станет доступен в браузере по адресу: http://localhost:8501
На дэшборде в реальном времени будут отображаться поступление новых заявок, оценка точности модели и распределение тикетов по отделам.
